## 1. Project Goal

The goal of this group project is to provide **hands-on, end-to-end experience in software engineering and project management** within a realistic team-based development setting.

Through this project, you are expected to practice **much more than coding**. You will learn how to:

- Engineer a software system **systematically**
- Manage a project **effectively**
- Collaborate **professionally**
- Communicate technical work **clearly**

This project simulates a **small software team**, where design decisions, planning, quality assurance, and teamwork matter as much as the final product.

### Scope and expectations

- The project is **not only about producing features**, but about demonstrating **engineering discipline throughout the development lifecycle**.
- Assessment emphasizes:
  - **how** the software is built  
  - **why** certain technical and managerial decisions are made  
  - **how well** the team collaborates and manages the project  
- A partially completed system with **strong engineering evidence** may score higher than a feature-rich system with **poor process or weak quality control**.

> Think like a software engineer and project manager, not just a programmer.

---

## 2. Repository and Collaboration

- Each team must use a collaborative development platform (i.e., **GitHub**).
- **One repository per team**.
- All members must contribute meaningfully through commits, pull requests, issues, and documentation.

---

## 3. Project Topic Requirements

### Mandatory components

Your project must include:

1. Clear stakeholders and requirements (functional and non-functional)
2. Architecture design with diagrams and rationale
3. Version control workflow
4. Continuous integration pipeline (build + test)
5. Testing strategy with evidence
6. Project management artifacts (issues, milestones, tasks)

### Not allowed

- Reuse of projects from other courses, internships, or competitions
- Projects completed before this course starts
- Projects copied or mostly taken from online tutorials or templates
- Illegal, plagiarized, or sensitive-data-collecting systems
- Toy projects with no engineering depth

External libraries and frameworks are allowed, but **all design and engineering decisions must be your own and developed during this semester**. External sources must be cited.

---

## Final Reminder

This project emphasizes **engineering quality, process, and teamwork** over sheer feature count.

You are not just building software, you are practicing being a software engineer.
